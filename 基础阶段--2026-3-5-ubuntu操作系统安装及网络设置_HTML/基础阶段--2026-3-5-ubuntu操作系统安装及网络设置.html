目录下会出现一个隐藏文件.1.c.swp
再vim 1.c 会弹出明确的提示（不是 “报错”，是正常的恢复提示），而非直接报错退出，Vim 会检测到 swap 文件并告诉你如何处理。

选项操作说明 适用场景
R（Recover） 恢复异常退出前未保存的修改想找回上次编辑的内容（最常用）
D（Delete） 删除 .1.c.swp 文件，正常打开 1.c确认上次的修改不需要保留
Q（Quit） 退出 Vim，不做任何操作暂时不想处理，先核对文件
O（Open Read-Only） 只读打开 1.c，不修改文件只想查看内容，不想误改

一、核心结论：选不同选项，后续提示与否的关键
只有 D 会立刻删除 swap 文件，其他选项（R/Q/O）都会保留 .1.c.swp，因此：
选择的选项后续是否还会出现提示？swap 文件是否保留？关键说明
D（Delete）❌ 不会（彻底解决）❌ 立刻删除选 D 后，swap 文件直接删，后续打开无提示
R（Recover）❌ 不会（恢复后自动删）✅ 暂时保留，恢复并保存后自动删除选 R 恢复内容 → 正常 :wq 保存退出 → swap 文件自动删，下次打开无提示；若恢复后未保存就退出，swap 文件仍保留，下次仍提示
Q（Quit）✅ 仍会提示✅ 完全保留只是退出 Vim，swap 文件没动，下次打开 1.c 还会弹提示
O（Open Read-Only）✅ 仍会提示✅ 完全保留只读打开文件，swap 文件没动，下次打开仍弹提示

或者可以在终端手动删除swap文件rm .1.c.swp

第一步：安装 VMware 网络驱动组件
• 操作：在 C:\Program Files (x86)\VMware\VMware Workstation 目录下，右键 netadapter.inf、netbridge.inf、netuserif.inf 这几个文件，选择「安装」。
• 作用：这几个 .inf 文件是 VMware 网络服务的驱动安装信息文件，安装后 Windows 系统才能识别并加载 VMware 的桥接网络核心组件，为后续配置做准备。



右键依次点击安装


第二步：打开物理网卡的属性配置
打开控制面板->网络和Internet->网络连接
找到以太网
• 操作：打开「网络连接」窗口（Win+R 输入 ncpa.cpl），右键你的物理网卡（如「以太网 2」「以太网 3」或 USB 网卡）→ 选择「属性」。
• 作用：进入网卡的配置界面，准备给物理网卡添加 VMware 的桥接服务。




第三步：给物理网卡安装 VMware 桥接服务
• 操作：1. 在网卡属性窗口点击「安装 (N)...」→ 选择「服务」→ 点击「添加 (A)...」；
2. 点击「从磁盘安装 (R)...」，浏览到 VMware 安装目录，选择 netbridge.inf（桥接协议的驱动文件）；
3. 确认安装，完成后网卡属性里会出现「VMware Bridge Protocol」。

4. 作用：将VMware 桥接协议绑定到你的物理网卡上，让物理网卡和虚拟机的网卡能 “桥接” 在同一个局域网里。




这一系列操作的核心是修复 / 启用 VMware 的「桥接模式」网络：• 只有桥接模式生效后，虚拟机才能和宿主机（你的电脑）处在同一个局域网网段；
• 之后你才能给虚拟机设置静态 IP（和宿主机同网段，比如宿主机是 192.168.1.100，虚拟机设为 192.168.1.101），实现宿主机、虚拟机、局域网其他设备之间的互通（比如 SSH 远程连接、文件共享）。
简单说：先把 VMware 的桥接网络 “打通”，之后才能给虚拟机配置静态 IP 并正常联网。如果之前桥接模式失效（比如虚拟机连不上网），重装这些驱动和服务是最常见的修复手段。
