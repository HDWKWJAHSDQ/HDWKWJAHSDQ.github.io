二、char 的深度解析（数值范围）
char 分为 有符号（signed char） 和 无符号（unsigned char），默认由编译器决定（多数编译器默认 char = signed char）：
1. 无符号 char（unsigned char）
位数：8 位，所有位均为数值位，无符号位。
范围：0 ~ 2^8 - 1 → 0 ~ 255。
计算逻辑：
最小值：所有位为 0 → 0000 0000 = 0；
最大值：所有位为 1 → 1111 1111 = 255。
2. 有符号 char（signed char）
位数：8 位，最高位为符号位（0 = 正，1 = 负），剩余 7 位为数值位。
范围：-2^7 ~ 2^7 - 1 → -128 ~ 127（注意不是 -127~128！）。
计算逻辑：
最大值：符号位 0 + 数值位全 1 → 0111 1111 = 127；
最小值：符号位 1 + 数值位全 0 → 1000 0000 = -128（补码规则）。
❗ 常见误区：很多人会记错成 -127~128，核心原因是混淆了 “符号位” 和 “数值位” 的边界 ——128 超出了 7 位数值位的表示范围（7 位最大是 127），而 -128 是补码的特殊取值。


二进制 (2)、八进制 (8)、十进制 (10)、十六进制 (16) 转换全攻略
一、核心概念
进制基数：二进制（0-1）、八进制（0-7）、十进制（0-9）、十六进制（0-9, A-F，A=10、B=11…F=15）
位权规则：任意进制数的每一位数值 = 该位数字 × 基数 ^（位数 - 1）（位数从右往左数，从 0 开始）

1. auto：自动类型推断 ✨
• 作用：让编译器自动帮你推导变量的类型，不用手动写。
• 场景：类型很长、或者不想写重复代码时用。
• 示例：cpp运行auto x = 10; // 编译器推导出 x 是 int 类型
auto s = "hello"; // 编译器推导出 s 是 const char* 类型

• 注意：auto 必须在定义时就初始化，否则编译器没法推断类型。

2. decltype：获取表达式的类型
• 作用：拿到某个表达式 / 变量的类型，用来定义新变量。
• 和 auto 的区别：auto 是 “根据值推类型”，decltype 是 “根据表达式拿类型”。
• int x = 10;
decltype(x) y = 20; // y 的类型和 x 一样，是 int
decltype(x+1.5) z; // z 的类型是 double（因为 x+1.5 是 double）

3. nullptr：空指针常量 int* ptr = nullptr; // 明确表示这是一个空指针
if (ptr == nullptr) {
// 判断指针是否为空
}
• 作用：专门表示 “空指针”，替代 C 语言里的 NULL。
• 优势：比 NULL 更安全，不会被误当作整数 0。
• 示例：
• int* ptr = nullptr; // 明确表示这是一个空指针
if (ptr == nullptr) {
// 判断指针是否为空
}
4. std::initializer_list：初始化列表类型
作用：让函数 / 类可以接收 {} 包裹的一组值作为参数。
场景：比如用 {1,2,3} 直接初始化容器、或者自定义类。
示例：

#include <initializer_list>
#include <iostream>

void print(std::initializer_list<int> nums) {
for (auto n : nums) std::cout << n << " ";
}

int main() {
print({1, 2, 3, 4}); // 直接传 {} 列表
return 0;
}

5. std::tuple：元组类型
作用：可以把多个不同类型的值打包成一个 “组合变量”。
对比数组：数组只能存同类型，tuple 可以存 int、float、char 等混合类型。
示例：
#include <tuple>
#include <iostream>
int main() {
// 定义一个包含 int、float、char 的元组
std::tuple<int, float, char> t(1, 2.0f, 'a');
// 取值：std::get<索引>(元组)
std::cout << std::get<0>(t) << " "; // 输出 1
std::cout << std::get<1>(t) << " "; // 输出 2.0
std::cout << std::get<2>(t) << endl; // 输出 'a'
return 0;
}
