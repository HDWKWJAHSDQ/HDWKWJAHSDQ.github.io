这是在 Ubuntu 系统下执行 ulimit -a 命令后显示的当前 shell 进程资源限制，用于控制进程对系统资源的占用。

资源项 当前值 含义与用途
open files 1024 单个进程可打开的最大文件描述符数，常用于高并发服务（如 Nginx、Redis）调优
max user processes 15140 当前用户可创建的最大进程数，影响多线程 / 多进程程序的并发能力
stack size 8192 kB 进程栈空间大小，默认 8MB，递归过深或栈变量过大可能导致栈溢出
max locked memory 495076 kB 可锁定在物理内存的最大空间，用于实时性要求高的场景（避免内存交换）
core file size 0 核心转储文件大小限制，0 表示禁用 core dump（调试时需开启）
cpu time unlimited 进程可占用的 CPU 时间上限，unlimited 表示无限制
内容类型C89C99/C11示例关键说明
整数字面量（正数）合法合法int arr[5];最基础写法，长度必须≥0（0 长度数组仅 C99 支持，多用于结构体柔性数组）
宏定义常量合法合法#define N 10; int arr[N];宏是预编译期替换，等价于字面量，C89/C99 都支持
枚举常量合法合法enum {SIZE=8}; int arr[SIZE];枚举常量是编译期常量，C89/C99 均支持
普通 const 变量非法非法const int n=5; int arr[n];❗C 语言中const仅表示「只读变量」，不是编译期常量，不能用于数组长度
编译期常量表达式合法合法int arr[3+2];仅支持字面量 / 宏 / 枚举的算术运算（如10-5、SIZE*2），编译期可计算
普通变量（变长数组 VLA）非法合法int n=7; int arr[n];C99 新增变长数组，仅局部作用域有效；C11 可选支持（可通过宏__STDC_NO_VLA__判断）
空括号[]（初始化）合法合法int arr[] = {1,2,3};由初始化列表自动推导长度，C89/C99 都支持
空括号[]（仅声明）非法非法int arr[];无初始化列表时，编译器无法推导长度，始终非法
空括号[]（函数形参）合法合法void func(int arr[]);形参中arr[]等价于arr*，长度无意义，C89/C99 都支持

关键补充（C 语言特有）：
const 变量的坑：
很多新手误以为 C 语言中const int n=5;是常量，但它本质是「只读变量」，编译期无法确定其值（比如可通过指针间接修改），因此 int arr[n]; 在 C89/C99 中都非法（和 C++ 不同）。
变长数组（VLA）的限制：
仅能定义在局部作用域（函数内、代码块内），全局 / 静态数组（static int arr[n];）即使 C99 也非法；
长度一旦定义，运行时不能修改（比如n=8后，arr长度固定为 7，无法再变）；
不能用sizeof(arr)获取真实长度（编译期无法确定，sizeof返回int的大小）。
二、C 语言中数组的最大可定义大小（纯 C 场景）
数组最大长度完全取决于「存储位置」，C 语言中数组主要分 栈区、全局 / 静态区、堆区 三类，各自限制不同：
1. 栈区数组（局部数组，如函数内定义）
存储位置：程序的栈空间（默认大小很小，Windows 约 1MB，Linux 约 8MB）；
最大长度：受栈空间限制，且需考虑 int 类型大小（通常 4 字节）。
示例：Linux 下栈区约 8MB=810241024=8388608 字节，int 占 4 字节，则栈区int数组最大约 8388608/4 = 2097152 个元素（约 200 万）；实际建议不超过 10 万，否则易触发「栈溢出（stack overflow）」。
示例（栈溢出风险）：
#include <stdio.h>
int main() {
// 风险：约4MB（100万*4字节），接近栈上限，可能崩溃
int arr[1000000];
return 0;
}

2. 全局 / 静态数组（static 或函数外定义）
存储位置：程序的全局 / 静态区（.data/.bss 段），空间大小远大于栈区（取决于系统内存）；
最大长度：理论上受操作系统虚拟内存限制（32 位系统约 4GB，64 位系统几乎无上限），实际受可用物理内存限制。
示例：
// 全局区数组，可定义更大（比如1000万int元素，约40MB）
int arr[10000000];
int main() {
// 静态区数组，和全局区一样
static int arr2[20000000];
return 0;
}
3. 堆区数组（动态分配，malloc/calloc）
存储位置：程序的堆空间（最大可分配内存接近系统总内存）；
最大长度：是 C 语言中能定义的最大数组，仅受系统物理内存 + 虚拟内存限制；
示例（推荐：大数组用堆区）：
#include <stdio.h>
#include <stdlib.h>
int main() {
// 动态分配1亿个int元素（约400MB），堆区可支持
int *arr = (int*)malloc(100000000 * sizeof(int));
if (arr == NULL) { // 必须检查分配是否成功
printf("内存分配失败\n");
return 1;
}
// 使用数组...
free(arr); // 用完必须释放
return 0;
}
特殊限制：0 长度数组
C99 支持「0 长度数组」（柔性数组），仅能定义在结构体末尾，无实际存储，用于动态扩展：
struct Data {
int len;
int arr[0]; // 0长度数组，仅占指针位置，无实际空间
};
// 动态分配时扩展长度
struct Data *p = (struct Data*)malloc(sizeof(struct Data) + 100*sizeof(int));
p->len = 100;
p->arr[0] = 1; // 可用arr[0]~arr[99]
总结
C 语言中数组[]里可写内容：C89 仅支持编译期常量（字面量 / 宏 / 枚举），C99 新增局部变长数组（变量长度），空括号仅初始化 / 函数形参时合法；
数组最大长度：栈区（小，MB 级）< 全局 / 静态区（大，几十～几百 MB）< 堆区（最大，GB 级），大数组优先用malloc分配堆内存；
核心坑点：C 语言中const变量不是编译期常量，不能用于数组长度；变长数组仅局部有效，且不能用sizeof获取长度。

srand(数);srand括号里的数会影响输出的随机数是否每次都一样
rand() 是伪随机数生成器，默认种子为1，所以不调用srand()时每次结果都相同；
srand(数) 的参数是随机数种子：种子固定→随机序列固定，种子变化→随机序列变化；仅需调用一次。

在C语言中，整型表达式的形式可以是以下几种：
1. 简单的整型变量，比如 x 或 y。
2. 整型常量，比如 5 或 -10。
3. 由整型变量和常量以及整型运算符（+、-、*、/等）组成的表达式，比如 x + 3 或 y * 2。
总的来说，整型表达式由整型变量、整型常量和整型运算符组成。
常量表达式就是在程序编译阶段就能确定最终值的表达式，它的值在程序运行过程中完全不会改变。
因为我们说数组大小一旦定义就不能改变，所以说数组的大小不能是变量

C 语言一维数组初始化核心规则
语法格式：类型 数组名[长度] = {值1, 值2, ...};，必须用大括号包裹初始值。
长度推断：如果省略 [长度]，则长度由初始化列表的元素个数决定。
不完全初始化：如果初始值个数少于数组长度，剩余元素自动初始化为 0（数值类型）或 '\0'（字符类型）。
空初始化：{} 在标准 C 中是非法的，不能用于初始化。
表达式允许：初始化列表中的值可以是常量表达式（如 10*1），编译器会在编译期计算其值。
