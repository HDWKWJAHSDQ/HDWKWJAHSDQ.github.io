面试题：
浮点数 与 "零" 比较？
1. 核心结论
不能直接写 if (num == 0) —— 浮点数（float/double）因二进制存储精度问题，哪怕理论值是 0，实际可能是0.0000000001这类极小值，直接用==会误判。
2. 正确写法（一行搞定）
#include <math.h> // 必须加这个头文件

// double类型（常用）：判断绝对值 < 1e-9
if (fabs(num) < 1e-9) {
// 认为num等于0
}

// float类型：判断绝对值 < 1e-6
if (fabsf(num) < 1e-6) {
// 认为num等于0
}
3. 总结
别用== 0，用fabs(数) < 很小的数；
double 用1e-9，float 用1e-6，记这两个值就够。

常用预处理指令：
指令核心功能
#include引入头文件（把指定头文件内容嵌入当前代码），如#include <stdio.h>引入标准输入输出声明
#define宏定义：替换文本 / 定义常量 / 宏函数，如#define PI 3.14（常量）、#define MAX(a,b) ((a)>(b)?(a):(b))（宏函数）
#ifdef/#ifndef/#endif条件编译：满足条件才编译某段代码，如#ifdef DEBUG（定义了 DEBUG 才编译后续代码），用于调试 / 跨平台开发
#undef取消已定义的宏，如#undef PI（之后 PI 宏失效）
#pragma向编译器发特殊指令（编译器相关），如#pragma warning(disable:4996)（关闭 VS 的 4996 警告）、#pragma pack(1)（设置内存对齐）
#error强制触发编译错误并输出指定提示，如#error "必须定义XX宏"，用于检查编译条件
#line修改当前代码的行号 / 文件名（调试用），如#line 100 "test.c"（后续代码行号从 100 开始，文件名显示 test.c）

内存？
虚拟内存 物理内存

1. 核心概念：虚拟内存 vs 物理内存
• 虚拟内存：每个进程拥有独立的 4GB（2³² 字节）虚拟地址空间（地址范围：0x00000000 ~ 0xffffffff），进程以为自己独占整个内存，这是操作系统抽象出来的逻辑地址，并非真实硬件内存。
• 物理内存：就是我们常说的内存条，是真实的硬件存储。
• 操作系统：负责通过 ** 内存管理单元（MMU）** 和页表，将虚拟地址映射到物理内存地址，实现进程间内存隔离与内存复用。



内存区域地址范围（32 位 Linux）存放内容生长方向管理方式生命周期读写权限
代码段（Text）低地址起始处程序的二进制机器指令、常量字符串（如 "hello"）固定操作系统加载时分配整个程序运行期只读（防止篡改）
数据段（Data）代码段上方全局变量、静态变量（static）- 已初始化：int a = 10;- 未初始化：int b;固定编译时确定，运行时分配整个程序运行期可读可写
堆（Heap）数据段上方动态分配的内存（malloc/calloc/realloc、new）向上（低→高）程序员手动管理（free/delete）手动分配 / 释放前可读可写

const 修饰的变量
const 是 C 语言中用来定义只读变量的关键字，核心作用是限制变量的值不能被修改，但它的具体行为（比如变量存在哪、是否真的 “不可改”）会根据变量的定义位置不同而有差异，我帮你拆解清楚：
一、const 修饰变量的核心规则
const 的字面意思是 “常量”，但在 C 语言中它修饰的是只读变量（不是真正的 “常量”，比如不能用来定义数组长度 int arr[const_a];，C99 前会报错），核心特点：1. 变量的值不能通过普通赋值修改（编译阶段就会报错）；
2. 变量的存储位置由定义位置决定（栈 / 数据段）；
3. 可以通过指针 “绕过” 限制修改（不推荐，属于未定义行为）。

二、不同位置的 const 变量（结合内存分段）

1. 全局 const 变量（最常用）
• 存储位置：代码段（Text）的只读区域（和常量字符串放一起）；
• 特点：真正的只读，操作系统会保护这块内存，强行修改会触发段错误（Segment Fault）。
c运行#include <stdio.h>

// 全局 const 变量：存在代码段（只读）
const int GLOBAL_CONST = 100;

int main() {
// 错误：编译阶段就会报错（assignment of read-only variable 'GLOBAL_CONST'）
// GLOBAL_CONST = 200;

// 尝试用指针修改（会编译通过，但运行时段错误）
int *p = (int *)&GLOBAL_CONST;
*p = 200; // 运行报错：Segmentation fault (core dumped)

return 0;
}

2. 局部 const 变量（函数内）
• 存储位置：栈（和普通局部变量一样）；

三、const 的常见使用场景
1. 保护变量不被误修改：比如函数参数用 const，防止函数内修改传入的变量（如 void print(const char *str);）；
2. 替代宏定义：相比 #define NUM 10，const int NUM = 10 有类型检查，更安全；
3. 修饰指针：区分 “指针只读” 和 “指向的内容只读”（高频考点）：• const int *p：p 指向的内容只读（不能通过 p 修改值），p 本身可以指向其他地址；
• int *const p：p 本身只读（不能改指向），但指向的内容可以修改；

在 C 语言中，有符号整数在内存中以补码形式存储，因此 a 保存的是 -5 的补码。
以 8 位有符号 char 为例计算：
5 的原码：0000 0101
-5 的原码：1000 0101（最高位 1 表示负数）
-5 的反码：1111 1010（原码除符号位外取反）
-5 的补码：1111 1011（反码 + 1）
32 位 int 的补码是在 8 位基础上高位全补 1，最终内存中存储的是补码形式。

2. 宽度（width）：指定输出的最小字符数
宽度是一个非负整数，若实际数字的字符数小于宽度，用空格 / 0（加0标志时）填充；若超过宽度，按实际长度输出。• 直接写数字：%5d（宽度 5）示例：printf("%5d", 123); → 输出 123（前面 2 个空格，总长度 5）
• 用*动态指定：%*d（宽度由后续参数传递）示例：printf("%*d", 5, 123); → 效果同上（5 是宽度参数）

3. 精度（.precision）：指定整数的最小位数（不足补 0，超过按实际）
精度以.开头，后跟数字 /*，仅对整数的 “位数” 生效，优先级高于0标志。• 直接写数字：%.3d（至少 3 位）示例：printf("%.3d", 12); → 输出 012（补前导 0 到 3 位）；printf("%.3d", 1234); → 输出 1234（超过 3 位，按实际）
• 用*动态指定：%.*d（精度由后续参数传递）示例：printf("%.*d", 3, 12); → 输出 012（3 是精度参数）
• 特殊情况：精度为 0 时，若数字是 0，输出为空 → printf("%.0d", 0); → 无输出。

4. 长度修饰符：匹配变量的整数类型（避免输出错误）

sizeof 是 C 语言的单目运算符（非函数），编译期计算操作数的内存字节数，不执行操作数本身，核心用法和字节大小如下：
用法含义字节大小 / 核心规则示例（32 位 / 64 位）
sizeof (数据类型)计算指定数据类型的内存大小基础类型大小固定（浮点 / 字符 / 短整型），长整型 / 指针随系统变化：• char：1• short：2• int：4・long：4（32 位）/8（64 位）• long long：8• float：4• double：8sizeof(int) → 4sizeof(double) → 8
sizeof (变量名)计算变量占用的内存大小等价于「变量对应数据类型的 sizeof」，和变量值无关int a=10; sizeof(a) → 4char c='a'; sizeof(c) → 1
sizeof (数组名)计算整个数组的总字节数总字节数 = 数组元素个数 × 单个元素字节数⚠️ 数组名退化为指针时（如传参），sizeof 计算的是指针大小int arr[5]; sizeof(arr) → 5×4=20（传参后：func(int arr[]) 中 sizeof(arr) → 4/8）
sizeof (指针)计算指针变量的内存大小和指向的类型无关，仅和系统位数有关：・32 位系统：所有指针均为 4 字节・64 位系统：所有指针均为 8 字节int *p; sizeof(p) → 4（32 位）/8（64 位）char **pp; sizeof(pp) → 4/8
sizeof (结构体)计算结构体总字节数需考虑内存对齐（编译器默认按最大成员对齐），总大小是最大成员字节数的整数倍struct S{char a; int b;};sizeof(S) → 8（char 占 1，对齐补 3，int 占 4，总计 8）
sizeof (常量)计算常量对应数据类型的字节数等价于常量类型的 sizeof，和常量值无关sizeof(10) → 4（10 是 int 型）sizeof(3.14) → 8（3.14 是 double 型）sizeof('a') → 4（C 语言中字符常量是 int 型）
sizeof(void)无意义（非法）void 是 “无类型”，无法计算大小；但sizeof(void*)合法（指针大小：4/8）sizeof(void) → 编译报错sizeof(void*) → 4（32 位）/8（64 位）

二、C 语言浮点数「e/E 计法」的核心规则（新手友好版）
科学计数法的本质是用 e/E 表示 “×10 的 n 次方”，格式必须满足：
[数字部分] e/E [整数指数部分]
• 数字部分：• 可以是「整数」（如 123）、「小数」（如 123.45、123.、.45），但不能为空；
• 示例：123e4（数字部分是整数 123）、123.45e-6（数字部分是小数 123.45）、123.e7（数字部分是 123.，合法）、.45e8（数字部分是.45，合法）。

• e/E：• 大小写均可（e 和 E 等价）；
• e/E 前后必须有合法数字，不能单独出现，也不能只在一侧有数字。

• 指数部分：• 必须是整数（可正可负，正号可省略），不能是小数、不能为空；
• 示例：123e3（指数 3，合法）、123e-4（指数 - 4，合法）、123E+5（指数 + 5，合法）；
• 反例：123e3.4（指数是小数，非法）、123e（指数为空，非法）。
